# Controller & Frontend Flow

## User Flow Overview

```
User on Campaign Review page
         │
         ▼
    Clicks "Deploy" button
         │
         ▼
Frontend: POST /api/v1/deploys (creates Deploy record)
         │
         ▼
Frontend: Redirect to /projects/:uuid/launch_deployment
         │
         ▼
Rails: ProjectsController#launch_deployment
         │
         ▼
Rails: Render Launch page with @project.to_launch_json
         │
         ▼
Frontend: Receives deploy in props, starts Langgraph flow
         │
         ▼
User can reload page → Same deploy continues
```

## Frontend: Deploy Button Action

When user clicks "Deploy" on the campaign review page:

```typescript
// In Campaign/Review component or similar

const handleDeploy = async () => {
  // 1. Create Deploy record
  const response = await fetch(`/api/v1/deploys`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ project_id: project.id })
  });

  const { id: deployId } = await response.json();

  // 2. Redirect to deployment page
  router.visit(`/projects/${project.uuid}/launch_deployment`);
};
```

## Rails Controller

The `launch_deployment` action is dynamically generated:

```ruby
# rails_app/app/controllers/projects_controller.rb

# This is auto-generated by WorkflowConfig
# WorkflowConfig.substeps_for("launch", "launch") returns ["settings", "review", "deployment"]
WorkflowConfig.substeps_for("launch", "launch").each do |substep|
  define_method("launch_#{substep}") do
    render inertia: "Launch",
      props: @project.to_launch_json,
      layout: "layouts/webcontainer"
  end
end
```

## Updated Serialization

Modify `to_launch_json` to include the active deploy:

```ruby
# rails_app/app/models/concerns/project_concerns/serialization.rb

def to_launch_json
  project = Project.with_launch_relations.find_by(id: id)

  to_ad_campaign_json.merge!({
    # Legacy: website deployment status
    deployment: project.website.respond_to?(:deployment) ? project.website&.deployment&.as_json : nil,

    # New: Active deploy record (latest in-progress or most recent)
    deploy: project.active_deploy&.as_json
  })
end
```

## Frontend: Receiving Deploy Props

```typescript
// Launch.tsx or similar

interface LaunchProps {
  project: Project;
  deploy: Deploy | null;  // Active deploy from server
  // ... other props
}

const Launch: React.FC<LaunchProps> = ({ project, deploy }) => {
  // If deploy exists, continue with it
  // If not, show "Deploy" button to create one

  useEffect(() => {
    if (deploy && deploy.status === "pending") {
      // Start Langgraph flow with this deploy
      startDeployFlow(deploy.id);
    }
  }, [deploy]);
};
```

## Page Reload Behavior

When user reloads the page:

1. Rails calls `@project.to_launch_json`
2. `active_deploy` returns the latest in-progress or most recent deploy
3. Frontend receives the same deploy in props
4. Langgraph continues from its checkpoint state

```ruby
# rails_app/app/models/project.rb

def active_deploy
  # First: any in-progress deploy
  # Second: most recent deploy
  deploys.in_progress.order(created_at: :desc).first ||
    deploys.order(created_at: :desc).first
end
```

## Workflow Step Tracking

Update workflow when entering deployment step:

```ruby
# rails_app/app/controllers/projects_controller.rb

define_method("launch_deployment") do
  # Update workflow to track current step
  @project.current_workflow.update!(step: "launch", substep: "deployment")

  render inertia: "Launch",
    props: @project.to_launch_json,
    layout: "layouts/webcontainer"
end
```

## Routes

No new routes needed - the dynamic method generation handles `/projects/:uuid/launch_deployment`:

```ruby
# rails_app/config/routes.rb

resources :projects, param: :uuid do
  # These are dynamically generated for each substep
  member do
    get :launch_settings
    get :launch_review
    get :launch_deployment  # <-- This one
  end
end
```
